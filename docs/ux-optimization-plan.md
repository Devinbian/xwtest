# 体验优化优先级清单（持续更新）

> 业务约束：本项目不展示价格/交期/质保/现货/服务承诺等固定信息（均需咨询后确定）。页面目标是引导用户通过电话/邮件/微信咨询；不提供在线提交需求功能。

## 🔴 立即修复（Critical）

1. 统一色彩系统：删除冲突色（例如轮播/按钮使用与主色不一致的蓝色），统一用主色体系（主色绿及其中性色/透明态）。
2. 修复移动导航：滚动锁定、修复高度计算，避免穿透与抖动。
3. 增大触摸目标：所有交互按钮至少 48×48px。
4. 添加键盘导航：所有交互元素可键盘访问（focus 样式清晰）。
5. 添加 ARIA 标签：提升无障碍可用性（搜索、筛选、弹窗、导航等）。

## 🟡 高优先级（High）

6. 实施 8px 间距系统：用 token 统一布局间距与内边距，减少“随机像素”。
7. 统一响应式断点：`640/768/1024/1280px`，并在关键布局处收口。
8. 建立字体比例系统：统一字号、行高、字重，降低信息层级噪音。
9. 简化导航菜单：减少层级与重复入口，移动端优先清晰“找产品/找服务/咨询”路径。
10. 优化图片加载：真正懒加载 + WebP/AVIF（保留 JPG 兜底），缩略图占位避免跳动。

## 🟢 中优先级（Medium）

11. 改进产品卡片（以“咨询”为导向）
   - 不展示价格；卡片文案强调“咨询可获取：价格/交期/质保/是否现货/服务承诺”。
   - 标签/规格更精细：默认一行保持一致，超出内容提供“更多/收起”展开。
   - 标题/摘要视觉统一：标题单行省略并提供 hover tooltip；摘要限制行数避免卡片高度不一致。
12. 优化动画性能：仅动画 `transform/opacity`，并支持 `prefers-reduced-motion`。
13. 咨询上下文自动带出：在咨询弹窗展示当前产品信息 + 当前筛选条件 + 页面链接（用于用户复制/描述，非在线提交）。
14. 添加空状态设计：提供“未找到产品”提示与快速清理筛选入口。
15. 添加深色模式：基于 token 快速切换（可选）。
16. 改进筛选体验：显示“已选筛选项”（可单项移除/一键清空），并保证 URL 同步与可分享。

## 🔵 低优先级（Low）

17. 添加微交互动画：hover/press 的细节反馈（不影响性能）。
18. 实施渐进式展示：首屏优先，后续模块按需渲染。
19. 添加产品对比功能（可选）：若业务后续需要再做。
20. 优化页脚布局：更清晰的联系方式与咨询入口。
21. 添加面包屑导航：提升当前位置感知（可选）。

## 💡 建议实施节奏

1. 第一周：色彩系统 + 移动端导航（Critical）
2. 第二周：设计系统（间距/字体/断点）（High）
3. 第三周：无障碍（键盘/ARIA）+ 筛选体验（Medium）
4. 第四周：性能优化（图片/动画）（High/Medium）
5. 后续：按业务优先级逐步推进中低优先级项
